# PID平衡车增加循迹模块 — 完整教案

## 一、概述

本教程讲解如何在已有的 STM32 PID 自平衡小车基础上，增加 TCRT5000 五路红外循迹模块，实现沿黑色轨迹线自动循迹行驶。涵盖硬件接线、代码架构、核心算法、参数调试全过程。

### 前置知识
- STM32F10x 基础外设（GPIO、定时器、中断）
- PID 控制原理（比例、积分、微分）
- 编码器测速原理

### 工程结构
```
6-2平衡车程序优化-增加寻迹模块/
├── User/
│   ├── main.c       ← 主逻辑 + PID控制 + 循迹控制
│   ├── PID.c/.h     ← PID算法实现
├── Hardware/
│   ├── Track.c/.h   ← 循迹传感器驱动（新增）
│   ├── Motor.c/.h   ← 电机PWM控制
│   ├── Encode.c/.h  ← 编码器读数
│   ├── PWM.c/.h     ← TIM2 PWM输出
│   ├── MPU6050.c/.h ← 姿态传感器
├── Basic/
│   ├── Timer.c/.h   ← TIM1 1ms定时中断
```

## 二、硬件资源

### 2.1 TCRT5000 五路循迹模块

| 参数 | 值 |
|------|------|
| 型号 | TCRT5000 五路红外反射传感器 |
| 输出 | 数字信号（0/1），**黑线→LOW，白面→HIGH** |
| 传感器间距 | S1-S2: 22mm, S2-S3: 24.5mm, S3-S4: 24.5mm, S4-S5: 22mm |
| 总跨度 | 93mm |
| 检测距离 | 8~15mm（需调电位器校准） |
| 供电 | 3.3V / 5V |

> ⚠️ **关键注意**：传感器超出检测范围（离地太远）时输出 LOW（与黑线相同），会被误判为黑线。平衡车前后倾斜时传感器离地高度会变化，必须在**正常平衡姿态下**校准。

### 2.2 引脚分配

| 传感器 | STM32引脚 | 说明 |
|:------:|:---------:|------|
| S1（最左） | PA9 | 原 USART1_TX，已释放 |
| S2（左中） | PA10 | 原 USART1_RX，已释放 |
| S3（中间） | PA11 | 空闲引脚 |
| S4（右中） | PA12 | 空闲引脚 |
| S5（最右） | PC15 | **备份域引脚**，需特殊处理 |

**PC15 特殊处理原因**：PC14/PC15 默认与 LSE（32.768kHz 晶振）关联，要作为普通 GPIO 使用，必须在初始化时：
1. 使能 PWR 和 BKP 时钟
2. 解锁备份域写保护
3. 关闭 LSE 振荡器
4. 禁用 PC13 侵入检测和 RTC 输出

### 2.3 实物接线图

```
TCRT5000模块              STM32最小系统板
  ┌──────┐               ┌──────────┐
  │ S1   ├───────────────┤ PA9      │
  │ S2   ├───────────────┤ PA10     │
  │ S3   ├───────────────┤ PA11     │
  │ S4   ├───────────────┤ PA12     │
  │ S5   ├───────────────┤ PC15     │
  │ VCC  ├───────────────┤ 3.3V     │
  │ GND  ├───────────────┤ GND      │
  └──────┘               └──────────┘
```

### 2.4 原有外设资源（不变）

| 外设 | 功能 | 参数 |
|------|------|------|
| TIM1 | 控制中断 | 72MHz / 72 / 1000 = 1ms |
| TIM2 | 电机PWM | 72MHz / 36 / 100 = 20kHz |
| TIM3 | 左轮编码器 | 编码器模式 TI12，4倍频 |
| TIM4 | 右轮编码器 | 编码器模式 TI12，4倍频 |
| GPIOB 12~15 | 电机方向 | 推挽输出 |

## 三、代码实现 — 循迹驱动层（Track.c/h）

### 3.1 文件结构

新建 `Hardware/Track.c` 和 `Hardware/Track.h`，包含：
- 5 个传感器 GPIO 初始化
- 单传感器读取函数
- 位掩码读取函数
- **加权重心法位置计算**（核心算法）

### 3.2 GPIO 初始化

```c
void Track_Init(void)
{
    // 1. 使能 GPIO 时钟
    RCC_APB2PeriphClockCmd(RCC_APB2Periph_GPIOA, ENABLE);
    RCC_APB2PeriphClockCmd(RCC_APB2Periph_GPIOC, ENABLE);

    // 2. PC15 备份域特殊处理
    RCC_APB1PeriphClockCmd(RCC_APB1Periph_PWR | RCC_APB1Periph_BKP, ENABLE);
    PWR_BackupAccessCmd(ENABLE);
    RCC_LSEConfig(RCC_LSE_OFF);
    BKP_TamperPinCmd(DISABLE);
    BKP_RTCOutputConfig(BKP_RTCOutputSource_None);

    // 3. PA9~PA12 上拉输入
    GPIO_InitTypeDef GPIO_InitStructure;
    GPIO_InitStructure.GPIO_Mode  = GPIO_Mode_IPU;
    GPIO_InitStructure.GPIO_Speed = GPIO_Speed_50MHz;
    GPIO_InitStructure.GPIO_Pin   = GPIO_Pin_9 | GPIO_Pin_10
                                  | GPIO_Pin_11 | GPIO_Pin_12;
    GPIO_Init(GPIOA, &GPIO_InitStructure);

    // 4. PC15 上拉输入
    GPIO_InitStructure.GPIO_Pin = GPIO_Pin_15;
    GPIO_Init(GPIOC, &GPIO_InitStructure);
}
```

### 3.3 传感器读取

```c
// 读取单个传感器（黑线=1，白色=0）
uint8_t Track_GetS1(void) {
    return (GPIO_ReadInputDataBit(TRACK_S1_PORT, TRACK_S1_PIN) == Bit_RESET) ? 1 : 0;
}

// 读取全部5路，返回5位掩码（bit0=S1 … bit4=S5）
uint16_t Track_Get(void) {
    uint16_t val = 0;
    if (Track_GetS1()) val |= 0x01;
    if (Track_GetS2()) val |= 0x02;
    if (Track_GetS3()) val |= 0x04;
    if (Track_GetS4()) val |= 0x08;
    if (Track_GetS5()) val |= 0x10;
    return val;
}
```

### 3.4 加权重心法（重点讲解）

**原理**：将 5 个传感器视为一维离散采样点，用质心公式计算黑线的等效横向位置：

```
               Σ (s[i] × w[i])
position = ────────────────────
                  Σ s[i]

s[i] ∈ {0,1}   — 第i个传感器检测结果
w[i] (单位mm)   — 传感器相对于S3的物理坐标
```

**传感器坐标**：
```
S1(-46.5)  S2(-24.5)  S3(0)  S4(+24.5)  S5(+46.5)
```

**代码实现**：
```c
float Track_GetPosition(void)
{
    uint8_t s[5] = {Track_GetS1(), Track_GetS2(), Track_GetS3(),
                    Track_GetS4(), Track_GetS5()};
    const float weights[5] = {-46.5f, -24.5f, 0.0f, 24.5f, 46.5f};

    float numerator = 0, denominator = 0;
    for (uint8_t i = 0; i < 5; i++) {
        if (s[i]) { numerator += weights[i]; denominator += 1.0f; }
    }

    if (denominator < 0.5f) return 999.0f;  // 全白 → 脱线
    if (denominator > 4.5f) return 0.0f;    // 全黑 → 认为居中

    return numerator / denominator;          // 正常计算
}
```

**计算示例**（视频演示用）：
```
S1  S2  S3  S4  S5     位置(mm)     含义
○   ○   ●   ○   ○       0.0        完美居中
○   ●   ●   ○   ○     -12.25      略偏左
●   ○   ○   ○   ○     -46.5       最左侧
○   ○   ●   ●   ○     +12.25      略偏右
○   ○   ○   ○   ●     +46.5       最右侧
○   ○   ○   ○   ○     999.0       脱线
●   ●   ●   ●   ●       0.0       全黑（弯道粗线）
```

## 四、代码实现 — 控制逻辑层（main.c）

### 4.1 总体控制架构

```
┌──────────┐    ┌──────────────┐    ┌──────────┐
│ 循迹传感器 │───→│ TurnPID      │───→│ DifPWM   │
│ (位置mm)  │    │ Target → Out │    │ (差速)   │
└──────────┘    └──────────────┘    └──────────┘

┌──────────┐    ┌──────────────┐    ┌──────────┐
│ 编码器   │───→│ SpeedPID     │───→│ 角度目标  │
│ (速度)   │    │ Target → Out │    │          │
└──────────┘    └──────┬───────┘    └──────────┘
                       │             ┌──────────┐
                       └────────────→│ AnglePID │
                                     │ → AvePWM │
                                     └──────────┘
```

### 4.2 三个 PID 控制器的角色

| PID | 更新周期 | Target | Actual | Out 用途 |
|-----|:--------:|--------|--------|----------|
| **AnglePID** | 10ms | SpeedPID.Out（目标角度） | 当前角度 | → AvePWM（平衡） |
| **SpeedPID** | 50ms | TrackSpeed 或摇杆值 | 平均速度 | → AnglePID.Target |
| **TurnPID** | 50ms | pos × kp（目标转向） | 左右差速 | → DifPWM（转向） |

### 4.3 循迹控制流程（50ms中断内）

```
┌─ 读5路传感器 ──────────────────────────────┐
│  Track_GetPosition() → pos (mm)            │
└────────────────────┬───────────────────────┘
                     ▼
           pos < 100.0f ?
          ┌──── YES ───┴─── NO ────┐
          ▼                        ▼
   ┌─ 死区处理 ──┐          脱线计数++
   │ |pos|<3mm   │          Phase 1: 向最后方向转
   │ → pos=0     │          Phase 2: 摆动扫描
   └──────┬──────┘          Phase 3: 原地自旋
          ▼
   ┌─ 弯道增强 ──┐
   │ pos>12mm    │
   │ → kp×2      │
   └──────┬──────┘
          ▼
   TurnPID.Target = pos × kp
   SpeedPID.Target = TrackSpeed
          ▼
   ┌─ PID更新 ──┐
   │ SpeedPID   │
   │ TurnPID    │
   └────────────┘
```

### 4.4 关键参数详解

```c
// ═══════ 循迹参数 ═══════
float TrackSpeed = 1.0f;    // 前进速度目标值
float TrackKp = 0.15f;      // 位置→转向目标的比例系数
                            // 最大偏移46.5mm → 目标值=46.5×0.15≈7
                            // TurnPID.Kp=5 → 输出≈35（未饱和）

#define CONF_TRACK_KP_CORNER_BOOST 2.0f
    // 弯道增强倍数。
    // |pos|>12mm时 kp 从 TrackKp 线性增至 2倍（30mm达峰值）。
    // 直线小偏移kp=0.15，弯道大偏移kp=0.30。

#define CONF_TRACK_DEAD_ZONE_MM 3.0f
    // 死区宽度。|pos| < 3mm 时认为居中，不做修正。
    // 防止传感器噪声引起频繁微调。

// ═══════ 转向PID（PD控制，Ki=0避免积分饱和）═══════
// Kp=5:  每个目标值单位产生5单位PWM差速
// Kd=1.5:微分阻尼，防止出弯过冲。越大转向越稳但响应越慢
// Ki=0:  循迹不需要积分，避免封闭轨道同向转弯的累积偏置

// ═══════ 角度PID（平衡环）═══════
// Kp=5:  每1°角度误差产生5单位PWM出力
// Ki=0.05:缓慢消除重心偏移等静差
// Kd=3:  微分阻尼，抑制前后摆动
// OutOffset=1:电机死区补偿（PWM<1电机不转，加1跳过）
```

### 4.5 脱线找回策略

| 阶段 | 时长 | 行为 |
|:----:|:----:|------|
| Phase 1 | ~400ms（8周期） | 向最后消失方向慢转，速度50% |
| Phase 2 | ~1.5s（30周期） | 每300ms摆动扫描，速度30% |
| Phase 3 | 1.5s以上 | 停止前进原地自旋找回 |

### 4.6 蓝牙调试指令

```

slider,TrackKp,0.12       // 调转向灵敏度
slider,TurnKd,2.0         // 调转向阻尼
slider,AngleKd,3          // 调平衡阻尼
slider,Offset,1            // 调电机死区补偿
track,1 / track,0         // 循迹开关
```

## 五、参数调试历程（视频讲解素材）

以下是调试过程中遇到的所有问题和解决方案，按出现顺序排列：

### 第1轮：首次能循迹
**问题**：循迹模块接入后完全不循迹
**分析**：`TrackKp=2.5` 过大（应为 0.1 量级），`TrackSpeed=2.0` 太快，`TurnPID.Ki=3` 积分过强
**解决**：TrackKp 2.5→0.2，TrackSpeed 2.0→1.0，TurnPID.Ki 3→0.05

### 第2轮：直角转弯过不去
**问题**：直线能循迹，90°圆角弯脱轨
**分析**：转向力度不足，传感器间距（22mm）> 轨迹带宽（18mm），
        黑线在传感器间隙时检测不到
**解决**：弯道自动增强（2x）、弯道平滑过渡（12~30mm线性）

### 第3轮：循迹"画龙"来回调节
**问题**：能跟住轨迹但左右频繁修正，不够丝滑
**分析**：PID 放大倍数过大，微小偏移被过度放大
**解决**：死区±3mm（小偏移不修正）、降低 TurnPID.Kp

### 第4轮：平衡抖动（走了弯路，重点讲）
**问题**：加了 Clamping Anti-Windup 后剧烈抖动
**分析**：Anti-Windup 检查 `p->Out`，而 `OutOffset` 把输出推高，
        导致平衡环积分被错误冻结 → 无法维持直立
**教训**：循迹不需要积分项（TurnPID.Ki=0），Anti-Windup 对平衡环有害

### 第5轮：转圈多了脱轨
**问题**：封闭矩形轨道转几圈后逐渐偏出
**分析**：AnglePID 积分累积 + 陀螺仪温漂，角度估计缓慢偏移
**解决**：AnglePID.Ki 减半、提高互补滤波加速计权重

### 第6轮：高速循迹来不及反应
**问题**：速度 1.2 时冲出轨迹来不及调整
**分析**：循迹控制只有 50ms（20Hz），50ms 内车已跑远
**教训**：编码器读数需要 50ms 累积才稳定，不能简单缩到 25ms

### 第7轮：换传感器模块
**问题**：传感器灵敏度不够，识别迟钝
**解决**：换带可调电位器的新模块，调整检测距离至 10~15mm

## 六、调试顺序建议

### Step 1 — 硬件验证
1. 下载程序，OLED 显示正常
2. 用手遮挡每个传感器，OLED 上对应位亮（1）/灭（0）
3. 循迹带放地上，确认各传感器能在正常高度识别黑白

### Step 2 — 基础循迹（S3单传感器测试）
1. 把循迹带放跑道中线，用手推车看是否跟线
2. 先调 `TrackKp`：太小不拐弯，太大来回摆。建议从 0.1 起步

### Step 3 — 死区测试
1. 让小车上线跑直道，观察是否频繁微调
2. 死区太小（1mm）→ 抖；太大（5mm）→ 直道跑偏
3. 3mm 是经验值

### Step 4 — 弯道调试
1. 跑矩形轨迹观察直角转弯表现
2. `CONF_TRACK_KP_CORNER_BOOST`：1.5~3.0 之间调
3. 弯道冲过头 → 增大 `TurnPID.Kd`

### Step 5 — 脱线找回
1. 人为把车拉出轨迹，观察找回行为
2. Phase 1 不够快 → 增大转向幅度（3.0f→4.0f）
3. Phase 2 扫不到 → 增大摆动幅度

### Step 6 — 速度优化
1. 蓝牙 `slider,TrackSpeed,1.2` 逐步加速
2. 太快脱轨 → 适当增大 `TrackKp` 或 `CONF_TRACK_KP_CORNER_BOOST`

## 七、常见问题 FAQ

**Q：传感器全部显示1（全黑）**
A：传感器离地太远超出检测范围，降低安装高度或调电位器

**Q：传感器都不亮**
A：检查供电（3.3V/GND）、检查GPIO配置是否和实际接线一致

**Q：循迹时左右剧烈摇摆**
A：`TurnPID.Ki` 是否非零？Ki 应设为 0。`TrackKp` 是否过大？

**Q：一开循迹就脱轨**
A：先用 OLED 观察传感器读数是否正常。控制周期内 Target 是否及时更新？

**Q：转圈多了慢慢偏出去**
A：检查 PID 有无积分累积（Ki）、互补滤波 Alpha 是否合适

**Q：换新循迹模块后代码需要改什么**
A：检查引脚是否兼容。如果引脚不同，改 `Track.h` 中的宏定义：
```c
#define TRACK_S1_PIN   GPIO_Pin_XX
#define TRACK_S1_PORT  GPIOX
```
再改 `Track.c` 中 `Track_Init()` 的 GPIO 配置。
